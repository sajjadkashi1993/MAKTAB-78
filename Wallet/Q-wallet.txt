
۲. میخواهیم کیف پول دیجیتالی خود را پیاده سازی کنیم:

حداقل شی های مورد نیاز : 
- پول با واحد های مختلف (دلار- یورو - تومان - ریال) 
- کیف پول (محلی برای نگهداری پول ها با شناسه ی اختصاصی و یکتا)
- کارت های اعتباری (بانکی، سوخت، تلفن یا حتی ملی )
- رسید ها (رسید های کارت های اعتباری)
- اطلاعات بازیابی (اطلاعاتی که در صورت عدم دسترسی با کد ، کیف پول قابل بازیابی باشد .)


نکات :
- در کیف پول شما هر ورود و خروجی باید ذخیره شود و رفتاری بر مبنای نمایش تمام تراکنش ها بصورت لیست (منظور لیست پایتون نیست!) محور وجود داشته باشد!
- کیف پول شما اطلاعات ورودی خاصی جز موارد بازیابی و کد دسترسی ندارد (کد توسط برنامه ساخته میشود! - میتوانید برای داده ی بازیابی پذیر از کد ملی استفاده کنید که نباید قابل تغییر باشد)
- رفتارهایی موجود باشد که موجودی کیف پول - موجودی تفکیکی هر مدل از پول - لیست رسید ها - حذف رسید دلخواه - کارت های اعتباری را ایجاب کند.


* امتیازی : برنامه را طوری بسازید که کلاس wallet امکان ساخت Instance را داشته باشد ولی هیچ instance برنگردد!
به مثال روبرو توجه کنید‌:

ali_wallet = Wallet(...)

قابل assign شدن نباشد و اررور دهد!
رفتار اینگونه باشد :
Wallet.create_wallet(...)  # Code = 121212
Wallet.get_wallet(121212) # return your wallet instance

در عمل اگر رفتاری برای دریافت تراکنش ها را در نظر بگیریم :
Wallet.get_wallet(121212) .get_transactions()
